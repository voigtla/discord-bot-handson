# Discord Bot 開発ハンズオン  
## 全10回ロードマップ（最終調整版）

本ロードマップは、  
メンタル系サーバー向け Discord Bot を題材に、

**「壊れず・覚えて・外とつながる」Botを  
段階的に育てていくための設計図**です。

各回は以下の方針で設計されています。

- 毎回やることは **1つだけ**
- 技術は **必要になった瞬間にだけ導入**
- 「作れる」より **「事故らない」を優先**

---

## 全体構成の考え方

このシリーズは、次の3フェーズに分かれています。

### フェーズ1：土台を作る（第1〜3回）
- 開発環境
- Git
- Botの最小構成

👉 **「動くものを作れる」状態になる**

---

### フェーズ2：壊れない設計を学ぶ（第4〜7回）
- メンタル系Botとしての制限
- DBの安全な使い方
- AI（Gemini API）の制御
- エラー・例外対応

👉 **「置いても大丈夫」なBotにする**

---

### フェーズ3：外につながる準備（第8〜10回）
- 倫理・安全・線引き
- デプロイの考え方
- 振り返りと次の分岐

👉 **「次に進める地図」を持って終わる**

---

## 各回の詳細

## 第1回（2025/01/17）  
### 開発環境セットアップ

**目的**  
今後の回で「環境が原因で詰まらない」状態を作る。

**主な内容**
- Node.js の確認
- VS Code の導入
- フォルダ構成の理解
- VS Code ワークスペースの使い方  
  （複数フォルダを左ペインに表示）

**補足**
- Cursor はオプション扱い
- 慣れている人のみ使用可

---

## 第2回（2025/01/30）  
### Git / GitHub 基礎（ハンズオン）

**目的**  
Git を「怖いもの」から「保存ボタン」に変える。

**主な内容**
- Git リポジトリ作成
- `npm init -y`
- add / commit / push
- README を編集して履歴に残す

**やらないこと**
- rebase
- 複雑なブランチ運用

---

## 第3回  
### Bot導入＋SQLiteで「1つだけ覚える」

**目的**  
Botが「動く」「覚える」を体験する。

**主な内容**
- discord.js の基本構造
- `/hello` コマンド実装
- SQLite に最小情報を保存

**重要な制限**
- 発言内容は保存しない
- user_id と時刻のみ扱う

---

## 第4回  
### メンタル系Botとしての最小実装（重要）

**目的**  
Botに「やらないこと」を教える。

**主な内容**
- ファイル分割  
  `index.js / db.js / responses.js`
- 返答文のテンプレ化
- 判断・助言・評価の禁止

**特徴**
- 説明時間は約10分
- 設計思想を固定する回

---

## 第5回  
### DBを使うが、評価しない

**目的**  
データを使っても事故らない感覚を作る。

**主な内容**
- `/count` コマンド追加
- SQLite から件数取得
- 事実のみを返す実装

**やらないこと**
- 比較
- スコア化
- 励まし・意味づけ

---

## 第6回  
### Gemini API 導入（自由生成しない）

**目的**  
AIを入れても制御できることを体験する。

**主な内容**
- Google アカウント準備
- Gemini API キー取得
- 文章整形専用での利用
- `aiFormatter.js` 追加

**重要な制限**
- ユーザー入力を渡さない
- 新しい文章を生成しない

---

## 第7回  
### エラー処理・例外・フォールバック設計

**目的**  
Botを「壊れにくく」する。

**主な内容**
- API エラー時の対応
- DB エラー時の対応
- フォールバック文言設計
- try / catch の整理

**思想**
- 正しく止まる
- 変なことを言わない

---

## 第8回  
### 安全・倫理・事故らない線引き

**目的**  
「やってはいけない」を共有知にする。

**主な内容**
- NG 実装例の確認
- 事故パターンの共有
- ルールを設計に戻す

**特徴**
- 設計・議論中心
- コードは最小限

---

## 第9回  
### このBotを「外に置く」としたら

**目的**  
本番運用の全体像を知る。

**主な内容**
- ローカルと本番の違い
- トークン管理
- デプロイ構成の考え方
- GCP / VPS 等の選択肢

**注意**
- 実デプロイ作業は行わない

---

## 第10回  
### 振り返り＋次の分岐点

**目的**  
学びを言語化し、次に進む準備をする。

**主な内容**
- できるようになったこと
- 危なかった設計
- このBotをどう使うか
- 次の学習ルート案内

**ゴール**
- 「Bot作れます」と言える
- 事故らない前提で説明できる

## このロードマップの特徴（まとめ）

- 途中参加・途中離脱を前提

- AIを過信しない設計
