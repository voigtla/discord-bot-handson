# Discord Bot 開発ハンズオン  
## 全10回ロードマップ（ハンズオン中心版）

このロードマップは、メンタル系サーバー向け Bot を題材に  
**「作る」より先に「壊さない」「戻れる」「渡せる」**を体験するための設計図です。

方針：

- 毎回のゴールは **1つだけ**
- 説明より **手を動かして成功させる**
- 秘密情報を貼らない（ログ/スクショ/資料も含む）
- 「やらないこと」を明示して迷子を防ぐ

---

## 全体構成（3フェーズ）

### フェーズ1：土台（第1〜3回）
👉 **Botが動くところまで到達する**

- 第1回：環境（Node / Git / VS Code）
- 第2回：Git = 保存ボタン化
- 第3回：Discordで /hello が返る（最小成功）

---

### フェーズ2：中身を育てる（第4〜7回）
👉 **覚える・読む・整える・壊れても安全に終わる**

- 第4回：SQLite に「1行だけ保存」
- 第5回：COUNTで「事実だけ返す」
- 第6回：Gemini を「文章整形専用」で導入（意味を増やさない）
- 第7回：エラー時の安全な終わり方（フォールバックを1か所に集約）

---

### フェーズ3：現実に耐える形にする（第8〜10回）
👉 **危ない実装を体験→止める→渡せる→分岐点に立つ**

- 第8回：わざと危ない返答を入れて、手で消して戻す（体験）
- 第9回：READMEを作り、READMEだけ見て復旧できる状態にする（引き渡し）
- 第10回：通し確認＋振り返り（できる/できないを言語化して次へ分岐）

---

## 各回の詳細（粒度：第1〜2回基準）

---

## 第1回：開発環境セットアップ
**ゴール（1文）**  
`node index.js` が動く

**やること（最小）**
- Node / Git / VS Code の導入と確認
- 作業フォルダを作り、VS Codeで開く
- ターミナルで `node index.js` を実行して文字が出る

**やらないこと**
- 理屈の深掘り
- 便利ツール前提（winget / brew等）

---

## 第2回：Git / GitHub 基礎（ハンズオン）
**ゴール（1文）**  
変更を commit / push して GitHub に残せる

**やること（最小）**
- リポジトリ化
- add / commit / push を体験
- README を更新して履歴で残す

**やらないこと**
- rebase / 複雑なブランチ運用

---

## 第3回：Discord Bot を起動する（/hello）
**ゴール（1文）**  
Discordで `/hello` を実行すると Bot が返事する

**やること（最小）**
- Developer Portal でアプリ作成→Bot作成→招待
- register（コマンド登録）と index（起動）を分けて理解
- /hello を返すだけ

**やらないこと**
- DB
- AI
- 複数コマンド

---

## 第4回：Botに「記憶」を持たせる（SQLite保存）
**ゴール（1文）**  
`/hello` のたびに SQLite に1件保存される

**やること（最小）**
- sqlite3 を追加
- 保存用の table を用意
- /hello で user_id と時刻だけ保存

**やらないこと**
- 読み出し
- 複雑なDB設計

---

## 第5回：保存したデータを「読む」（/count）
**ゴール（1文）**  
`/count` で自分の記録件数を数字で返せる（評価しない）

**やること（最小）**
- register に /count を追加
- DBから COUNT して返す（事実のみ）

**やらないこと**
- 比較
- スコア化
- 励まし・意味づけ

---

## 第6回：AIを「文章整形専用」で使う（Gemini）
**ゴール（1文）**  
AIを通しても意味が変わらない

**前提（必要なもの）**
- Googleアカウントでログインできる
- APIキーを取得して安全に管理できる（貼らない）

**やること（最小）**
- Gemini API を有効化しキー取得
- aiFormatter を追加
- Botの固定文だけを整形（ユーザー入力は渡さない）

**やらないこと**
- 自由生成
- 判断・助言・評価
- ユーザー入力のAI送信

---

## 第7回：エラーでも安全に終わる（フォールバック）
**ゴール（1文）**  
失敗しても固定文で終われる

**やること（最小）**
- 失敗時の文言を1か所に集約
- 失敗処理を safeReply に集約
- わざと失敗させて確認する（キーを一時的に外す等）

**やらないこと**
- 詳細エラー説明
- 自動再試行

---

## 第8回：わざと危ない実装 → 手で止める（体験）
**ゴール（1文）**  
危ない一文を入れて、違和感を見て、自分で消せる

**やること（最小）**
- /count に“余計な一言”をわざと入れる
- Discordで違和感を確認
- その一文を削除して元に戻す（コミットしない）

**やらないこと**
- 講義だけで終わる線引き
- README宣言だけ

---

## 第9回：他人に渡しても壊れない形にする（README作成）
**ゴール（1文）**  
READMEだけ見て Bot を復旧できる

**やること（最小）**
- README を作る（起動手順を固定）
- Botを止める → READMEだけ見て起動 → 触って確認

**やらないこと**
- “初期セットアップ”のやり直し（npm install等）
- デプロイ講義だけ

---

## 第10回：通し確認＋振り返り（分岐点）
**ゴール（1文）**  
自分ができること・次の選択肢を説明できる

**やること（最小）**
- 起動→/hello→/count を通す
- 何ができるBotか1文で言う
- 次の分岐（拡張/別Bot/寝かせる）を決める

**やらないこと**
- 新機能追加
- デプロイ実作業

---

## まとめ（このシリーズの到達点）

この10回の到達点は：

- Botが動く
- ちょっと覚える
- 事実だけ返す
- AIは整形専用で制御する
- 失敗しても安全に終わる
- READMEで他人に渡せる

派手ではないけど、**壊れにくく説明できる**Botを作れます。
