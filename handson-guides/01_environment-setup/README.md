## 🧰 使用するエディタについて（重要）

### 標準エディタ：VS Code

このハンズオンでは、**VS Code を標準エディタ**として進めます。

理由：
- 無料で安定している

- **初参加・これから学ぶ人 → VS Code 推奨**

## 4️⃣ 作業用フォルダを作る（git_practice）

今回は、**すべての練習を `git_practice` フォルダで行います**。

### フォルダ作成

Windows Terminal で以下を実行します。

```bash
mkdir git_practice
```

---

## 5️⃣ VS Code でフォルダを開く（重要）

### フォルダを開く方法

1. VS Code を起動
2. メニューから
   **ファイル → フォルダーを開く**
3. `git_practice` フォルダを選択
4. **フォルダーの選択** をクリック

左側（エクスプローラー）に
`git_practice` が表示されていればOKです。

---

## 6️⃣ 複数フォルダを扱う場合（ワークスペース）

今後、以下のような状況が発生します。

* 複数のリポジトリを同時に開きたい
* 参照用フォルダと作業用フォルダを並べたい

その場合は **ワークスペース機能**を使います。

### フォルダを追加する手順

1. VS Code メニュー
   **ファイル → フォルダーをワークスペースに追加**
2. 追加したいフォルダを選択

すると、左ペインに複数フォルダが並びます。

---

### ワークスペースを保存する（推奨）

1. **ファイル → 名前を付けてワークスペースを保存**
2. `.code-workspace` ファイルを保存

次回からは：

* VS Code でこのファイルを開くだけで
* 同じフォルダ構成が復元されます

👉 **複数フォルダを扱う場合は必ず保存してください**

---

## 7️⃣ 最初の動作確認（Node.js）

### ファイル作成

VS Code の左ペインで：

* `git_practice` を右クリック
* **新しいファイル**
* ファイル名：`index.js`

中身を以下にします。

```js
console.log("環境構築OK");
```

以下を実行：


`環境構築OK` と表示されれば成功です 🎉

## 🔹 補足：Cursor + AI を使った開発パターン（参考）

※ これは **オプショナル** です
※ VS Code 利用者は **読み飛ばしてOK**

### Cursor を使う場合の流れ

1. 作業ディレクトリを作成
2. Cursor でそのディレクトリを開く
3. モードを **Plan** に変更
4. 以下のプロンプトを入力

#### プロンプト例

```
/hello コマンドだけの Discord ボットを作って。
コメント、仕様書、すべて日本語で。
```

Cursor は以下のような計画を自動生成します。

* bot.js（メイン処理）
* .env.example
* .gitignore
* README.md（仕様書）
* package.json 更新内容

### 注意点

* 無料プランでは利用制限にすぐ到達します
* AIの出力は **そのまま使わず必ず読む**
* 「何が作られたか」を理解できない場合は使わないでください

👉 **Cursorは補助輪、VS Codeは基礎体力** という位置づけです。
