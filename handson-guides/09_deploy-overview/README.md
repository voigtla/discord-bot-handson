## 🎯 この回のゴール

この回のゴールは1つだけ。

> **「本番運用とは何か」を  
> 言葉で説明できるようになる**

---

## なぜローカル実行では足りなくなるのか

ここまでの Bot は、

- 自分の PC で動かす
- 自分が起動・停止する
- 自分が見ている前提

でした。

でも「外に置く」と、前提が変わります。

- 誰かがいつ叩くかわからない
- 自分が寝ていても動く
- 落ちたら誰も気づかない

---

## 「外に置く」と増える責任

Botを外に置くと、次の責任が発生します。

- トークンを漏らさない
- 勝手に止まらない
- 変なログを吐かない
- 更新時に事故らない

**機能の話ではありません。運用の話です。**

---

## 本番構成の最小イメージ

多くの場合、構成はこうなります。

```

Discord
↓
Bot（Node.js）
↓
DB（SQLite or 他）

```

これを、

- 自分の PC ではなく
- どこかのサーバーで
- 常時起動させる

---

## サーバーの選択肢（考え方だけ）

第9回では、**選択肢を知るだけ**です。

### 選択肢の例

- VPS（ConoHa / さくら / 他）
- クラウド（GCP / AWS / Azure）
- PaaS（Render / Railway など）

👉 どれを選んでも  
**考えることの本質は同じ**。

---

## 外に置くときに必ず考えること

### 1️⃣ トークン管理

- `.env` を Git に上げない
- サーバー側にだけ置く
- 人に共有しない

---

### 2️⃣ 起動と再起動

- Node をどう起動するか
- 落ちたらどうするか
- 再起動は自動か手動か

---

### 3️⃣ ログの扱い

- エラーはどこに出るか
- 誰が見るのか
- 見なくていいログは何か

---

### 4️⃣ 更新のしかた

- いつ止めるか
- どう差し替えるか
- 元に戻せるか

---

## 第7回・第8回が効いてくる理由

ここで、第7・8回が効きます。

- エラー時に暴れない
- 判断・助言をしない
- 危険な処理を持たない

👉 **だから「外に置ける」**
